git init: Iniciar repositorio vacio

git clone ruta_de_carpeta: clonar repositorio, es decir abrir uno existente, tipo, importar un repositorio

git log: permite ver los cambios en cada commit

git add: aniadir  archivos al area de preparacion que luego querr'e guardar su version (commit)

git commit -m "mensaje que yo quiera ponerle, ej> introduccion": ejecuta un commit, lo guarda

git status: verificar estado del repositorio 

git diff commit_id commit_id: ver diferencias en los archivos de los dos commits, el id es el numero largo despues de commit

cheatsheet git checkout commit_id: permite volver a la version que se indique

git tag -a nombre_del_commit: es para cambiarle el id al commit por un nombre mas entendible


una rama es una version alternativa  a la version original, sirve para ir avanzando en una version alternativa del archivo original hata esperar que se aprueben modificaciones por, por ejemplo, por el jefe de grupo, para esto se usa el comando git branch nombre_de_la_rama (nombre archivo paralelo)

para aniadir todos los archivos de la carpeta al repositorio utilzo git add .   el punto indica que todo lo que esta en la carpeta se aniade.

para unir el archivo principal (master) con el alternativo (rama o branch), tipeo git checkout master y luego git merge nombre_rama
